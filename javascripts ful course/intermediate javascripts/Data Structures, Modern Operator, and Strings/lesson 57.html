SETS

A set is a collection of unique values. This means that a Set can only contain distinct elements, and it automatically removes any duplicates. Sets are particularly useful for storing and managing a collection of unique items.

Javascript's Set object provides several features that make it useful data structure for managing collections of unique values.

1. Creating an Empty set Set

  • Syntax:
    • const mySet = new Set()

2. Creating a Set with initial Values

  • Syntax:
    • const mySet = new Set([1, 2, 3, 4, 5, 6])

3. Adding Values

To add a new value to Set the add method has to be used.

  • Syntax:
    • const mySet = new Set([1, 2, 3, 4, 5, 6])
      mySet.add(value)

4. Checking for a Value

To check if a value exists in Set, the has method has to be used.

  • Syntax:
    • const mySet = new Set([1, 2, 3, 4, 5, 6])
      mySet.has(value); // true

5. Clearing the Set

Clearing the Set is the method of removing all the values from the set, to implement that, the clear method has to be used

  • Syntax:
    • const mySet = new Set([1, 2, 3, 4, 5, 6])
      mySet.has(value) // Set { }

6. Getting the Size of the Set

Getting the Size of the Set is the method of checking the length of values in Set. It can be checked by using the size method.

  • Syntax:
    • const mySet = new Set([1, 2, 3, 4, 5, 6])
      mySet.size// 6

7. Iterating Over a Set

Iteration is allowed in Set by using the for...of loop method

  • Syntax:
    • const mySet = new Set([1, 2, 3, 4, 5, 6])

      for(const sets of mySet) {
          console.log(sets);
      }

8. Converting Between Sets and Arrays

A set can be converted to array and vice and versa.

  • Syntax:
    • const array = [1, 2, 3, 4, 5, 6]
      const mySet = new Set(array);

      const newArray = [...mySet];
      console.log(newArray);

PROS AND CONS OF SETS

Pros:

  1. Uniqueness: Ensures all elements are unique, preventing duplicate entries.
  2. Efficiency: Provides efficient methods for checking for checking existence, adding, and deleting elements, typically with average time complexity of 0(1) for these operations.
  3. Iteration: Supports easy iteration through elements using methods like for...of loops
  4. Size Property: Allows quick access to determine the number of elements in the set using the size property.
  5. Convenience Methods: Includes useful methods like add, delete, has, and clear.
  6. Order of Insertion: Maintains the order of insertion, which can be useful in certain scenarios.

Cons:

  1. Limited Features: Lacks some features found in arrays, such as direct access via indices and many array-specific methods like map, filter, and reduce.
  2. Weakest Limitations: If using Weakest, it only accepts objects as values and doesn't have methods to iterate or check the size.
  3. Memory Usage: May use more memory compared to arrays for storing the same number of elements, especially if the elements are primitive values.
  4. Compatibility: May not be supported in very old Javascript environments, though this is less of an issue with modern browsers and Node.js versions.
